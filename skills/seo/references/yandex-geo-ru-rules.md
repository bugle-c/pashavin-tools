# Russian SEO/GEO rules — Yandex + Google + AI engines

Канонический свод правил для русскоязычных сайтов под Яндекс, Google и AI-поисковики (Geo.ai, ChatGPT search, Perplexity). Источник — рабочая практика 2025-2026 + livestream-разбор от 2026-04-06 (`/tmp/pasha_uploads/seo-rules-2026-04-06_16-59-43.txt`).

Если эти правила противоречат общим скиллам (`seo-content`, `seo-page` и т.д.), для русскоязычных коммерческих проектов **приоритет у этого документа**.

## 1. Поведенческие сигналы — главное в Яндексе

- **Время на странице (dwell-time) — ранжирующий сигнал #1**. Если у конкурента 1:30, у вас 1:00 — конкурент всегда выше по этому запросу.
- **Глубина просмотра + возвраты в выдачу** — второй по силе сигнал. Чем меньше pogo-sticking, тем выше позиции.
- **Анимации / интерактив на главной**: пиксельные сценки, SVG-анимации, hover-эффекты на блоках. Цель — задержать взгляд на 10-20 секунд. Это даёт измеримый рост по позициям. JS-анимации канваса/SVG **нагрузку дают минимальную**, не бойтесь.
- **YouTube-плеер на странице** работает в RU-сегменте ненадёжно (заблокирован для части юзеров → ошибка авторизации → bounce). VK Video тупит. Лучше — собственный `<video>` с self-hosted MP4.

## 2. Объём контента (русскоязычные коммерческие страницы)

- **Минимум 1200-1500 слов на странице** для конкуренции в выдаче Яндекс/Google по коммерческим запросам.
- Это **не общая планка от Google** (тот говорит про "topical coverage floor"), а проверенный практикой минимум для русского рынка.
- Распределение: hero (≥150 слов) + 4-6 информативных блоков (по 150-300 слов) + FAQ-секция.
- Текста не хватает — добавлять в info-блоки, не в hero. Hero-блок плотным текстом убивает конверсию.

## 3. Стратегия запросов

- **Не лезть в широкие запросы** ("биткоин", "форекс", "обучение"). Там Wikipedia, биржи, информационные порталы с миллионными бэками. Дай бог 20-30 страница.
- **Длинный хвост — основа**: "обучение трейдингу с нуля", "купить диван угловой Москва", "вет клиника собака". 100-200 показов/мес с низкой конкуренцией — sweet spot.
- **Body-first selection**: первичный ключ выбирается так, чтобы он реально присутствовал в теле страницы, а не натянут сверху.
- Генерические порталы (Wiki, Дзен, VK, Avito, Habr, otzyv-сайты) **не считаются конкурентами** при оценке competition.

## 4. JSON-LD Schema.org (обязательно для Google AI Overviews и Я.Алисы)

Минимальный набор на коммерческой странице:

| Schema | Где |
|---|---|
| `Organization` или `Person` | На всех страницах (привязывает бренд к домену) |
| `BreadcrumbList` | На всех нелендинговых страницах |
| `WebPage` или `Article` | По типу страницы |
| `FAQPage` | На страницах с реальной FAQ-секцией (3+ вопросов) |
| `Course` | Для образовательных продуктов (даже бесплатных — хороший лид-магнит) |
| `Product` + `Offer` + `AggregateRating` | Карточки товаров e-commerce |
| `LocalBusiness` (или подкласс) | Локальный бизнес (вет.клиника, салон, ресторан) |

Без JSON-LD AI-движки (ChatGPT, Perplexity, Google AI Mode) **не цитируют страницу**, даже если она в топ-10.

## 5. Yandex trust paths — обязательно для коммерческих сайтов

Без этих сигналов Яндекс срезает позиции по коммерческим запросам, даже при идеальной on-page оптимизации:

1. **Cookies-плашка** для новых посетителей с явным согласием.
2. **Политика конфиденциальности** — отдельная страница `/privacy`.
3. **Политика обработки cookies** — отдельная страница (или раздел в /privacy).
4. **Юридические реквизиты владельца** в footer:
   - ИП / ООО / самозанятый — название юрлица
   - ИНН, ОГРН/ОГРНИП
   - Город регистрации
   - Email + телефон + Telegram
5. **Кастомная страница 404** с дизайном, а не дефолтная серверная (важно и для Google, и для Яндекса).
6. **Контактная страница** `/contacts` с реальными данными.
7. **Микроразметка владельца** (Organization / Person с `address`, `legalName`, `taxID`).

⚠ Если на одном лице несколько сайтов — **не дублируйте юридические данные**, иначе Яндекс склеит сайты в один и трафик распределится между ними вместо удвоения.

## 6. Яндекс.Метрика и Google Analytics

- **Яндекс.Метрика обязательна** — без неё для Яндекса вы "неавторизованный", ИКС/ТИЦ не растёт, доверие нулевое. Включать webvisor + clickmap + accurateTrackBounce.
- **Google Analytics можно ставить** — но при сборе любых данных подаётся **уведомление в РКН** (форма на портале РКН). Даже если просто счётчик без ПДн — формально требуется регистрация оператора ПДн. После уведомления — никаких проблем.
- Без счётчиков сайт для поисковика — анонимный, ранжирование занижается.

## 7. Изображения

- **Только свои фото / AI-генерация**. Чужие фото из интернета = риск штрафа от 10к до 250к ₽ за изображение (фотобанки активно мониторят).
- **AI-сгенерированные изображения пока разрешены**, бан в выдаче не прилетает (по состоянию на 2026-04). Закона против них нет.
- **Свои фотобанки / стоковые** — оплаченная подписка, лицензия на коммерческое использование.
- На каждой `<img>`: явные `width`+`height` (CLS), `loading="lazy"` (не hero), `decoding="async"`, осмысленный `alt` (описание содержания, не "image"), `title` для SEO.
- WebP/AVIF предпочтительнее PNG/JPEG.

## 8. GEO / AI-поисковая оптимизация

GEO (Generative Engine Optimization) — оптимизация под цитирование AI-движками. Появилось ~8 месяцев назад (середина 2025), модели об этом часто не знают — при работе с LLM явно говорить "Geo.ai/AI search optimization" + давать определение.

Что критично:
- **Definition-first intro** в первых 50-150 словах: "X — это ...". AI вытаскивает этот абзац как цитату.
- **Question-led H2** ("Что такое...?", "Как работает...?", "Сколько стоит...?") — ≥60% заголовков вопросами.
- **Self-contained answer blocks** 134-167 слов под каждым вопросом — оптимальная длина для цитирования.
- **Конкретные числа, даты, имена** в тексте — AI любит факты, не вату.
- **Server-side rendering** для ключевого контента (AI-краулеры **не выполняют JS**).
- **`/llms.txt`** в корне домена — структурированный гид для AI-краулеров.
- **`robots.txt`**: разрешить `GPTBot`, `OAI-SearchBot`, `ClaudeBot`, `PerplexityBot`. `CCBot` — по желанию (training crawler).

## 9. Внутренняя перелинковка

- 3-5 контекстных внутренних ссылок на 1000 слов.
- Анкоры — описательные, не "тут" / "здесь".
- На каждую новую страницу — ссылка с главной или из меню (никаких сирот).
- На карточках товара/услуги — блок "Похожие" / "Связанные".

## 10. Workflow создания страниц через AI (vibe-coding)

Из стрима 2026-04-06 — рабочий пайплайн в Cursor:

1. **Дробить слона**: hero → меню → блок 2 → блок 3. Не давать "сделай весь сайт" одним промптом — режется в контекст-токены, получается каша.
2. **Версионирование тем**: после каждой правки `theme-v1.zip, v2.zip…`. Сломалось — откат на пред. версию (не редкость).
3. **Модели по задачам**:
   - **Дизайн / визуал** — Gemini 3.x Pro (лучший в визуале)
   - **Код / сложные баги** — GPT-5.4 (обогнал Sonnet/Opus)
   - **Лёгкий код, быстрые правки** — Composer 2 (Kimi 2.5 + Cursor-допилка)
   - **SEO/GEO разметка** — Gemini (хорошо знает алгоритмы Google/Яндекс)
   - **Sonnet 4.6 / Opus 4.6** — отодвинуты, особенно Opus после введения 1M контекста заметно просел
4. **Cursor `.cursorrules` правило**: "никаких уточняющих вопросов пользователю, делай задачу end-to-end". Иначе LLM упирается в `if user wants X`.
5. **PageSpeed-фикс**: открыть `pagespeed.web.dev/<url>` → скопировать ВСЕ красные/оранжевые ошибки → отдать в Cursor одним промптом → пусть фиксит. Руками не лезть.
6. **Mobile-first проверка**: после каждой правки — `npm run dev` + DevTools → 375px + 390px viewport. Десктоп-only ломается на мобиле.

## 11. Специфика по типам сайтов

### Локальный бизнес (клиника, салон, мастерская)
- `LocalBusiness` schema с `address` (NAP — Name/Address/Phone должно совпадать на всех площадках: сайт, Я.Бизнес, 2GIS, Google Business)
- Я.Бизнес профиль обязателен, виджет "как добраться" на /contacts
- Отзывы (свои + интеграция с Я.Картами/Otzovik)
- Кейсы / до-после с реальными фото (не стоковые)

### Образовательный продукт / курс
- `Course` schema даже для бесплатных мини-курсов (литмагнит → доверие)
- Программа курса в виде `Syllabus` + дат
- Преподаватель → `Person` schema с `knowsAbout`, `alumniOf`

### E-commerce / маркетплейс
- `Product` + `Offer` (цена, наличие) + `AggregateRating`
- Хлебные крошки на всех уровнях категорий
- Каноникал на канонический URL (без `?utm`, `?sort` и т.д.)
- WordPress + S3 для хранения изображений (большие медиа-папки убивают сервер)

### Personal brand / portfolio
- `Person` schema с `sameAs` (соц-сети) + `knowsAbout`
- Кейсы как `WebPage`/`SoftwareApplication`, не `Article`
- Footer с юр.именем + Telegram (без ИНН — для физлица не обязательно)

## 12. Антипаттерны (часто видимые ошибки)

- ❌ Конструкторы (Tilda, WordPress без вмешательства) → нет JSON-LD, кэширование сломано, дубли в URL.
- ❌ Hero-блок с одной фразой и кнопкой — нечего ранжировать.
- ❌ Картинка вместо текста (особенно H1 на изображении) — поисковик не видит.
- ❌ Один и тот же `<title>` / `meta description` на всех страницах.
- ❌ `noindex` забыли убрать после dev — частая причина "сайт не в индексе".
- ❌ Sitemap.xml без даты обновления (`<lastmod>`) — Яндекс не понимает свежесть.
- ❌ Дубли через trailing slash, www/non-www, http/https без 301-редиректа.

```
[ ] Кастомная 404

[ ] JSON-LD: Organization + BreadcrumbList + (Page-type-specific)
[ ] Title 38-65 / Description 120-165 chars
[ ] H1 один, H2 question-led ≥60%, H3 строго под H2
[ ] 1200+ слов на коммерческих страницах
[ ] Definition-first intro 50-150 слов
[ ] Все <img> с width/height/alt/loading=lazy
[ ] PageSpeed Mobile ≥80, SEO ≥95 (зелёные)
[ ] sitemap.xml + robots.txt с Allow для GPTBot/ClaudeBot/PerplexityBot
[ ] /llms.txt в корне

```
